⚙ Setup Guide

Step 1 — Supabase DB

Run schema_v2.sql in Supabase SQL Editor. This creates all tables matching the APK's data model.

Step 2 — Edge Function

supabase functions new axesms-api
# paste edge-functions-v2.js content
supabase functions deploy axesms-api

Step 3 — Env Variables

SUPABASE_URL=https://xxx.supabase.co
SUPABASE_SERVICE_KEY=your_service_role_key
APK_BRIDGE_SECRET=your_secret_here
ADMIN_SECRET=your_admin_secret

Step 4 — Firebase Bridge

npm i firebase-admin @supabase/supabase-js dotenv
node firebase-bridge.js

APK Integration — How it works

The APK (com.rooted.panel) uses Firebase Firestore for realtime sync and a MySQL/HTTP backend for API calls. It syncs the server base URL via panel/config from Firestore. Your APK should call /apk/heartbeat every 30s, /apk/queue to get pending SMS, and /apk/delivery + /apk/incoming to report results. The Firebase Bridge handles the realtime sync automatically.
